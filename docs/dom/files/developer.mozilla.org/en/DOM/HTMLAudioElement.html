Introduced in Gecko 1.9.1

(Firefox 3.5 / Thunderbird 3 / SeaMonkey 2)

Introduced in HTML 5

The HTMLAudioElement interface provides access to the properties of <audio> elements, as well as methods to manipulate them. It's derived from the HTMLMediaElement interface; it's implemented by nsIDOMHTMLMediaElement.

For details on how to use the audio streaming features exposed by this interface, please see Introducing the Audio API Extension.

Properties

Name Type Description
mozCurrentSampleOffset Requires Gecko 2.0 Non-standard unsigned long long
 		Indicates the current offset of the audio stream that was created by a call to mozWriteAudio(). This offset is specified as the number of samples since the beginning of the stream.

Methods

Name & Arguments Return Description
Audio() HTMLAudioElement Constructor for audio elements. The preload property of the returned object is set to auto.
Audio(in DOMString src) HTMLAudioElement Constructor for audio elements. The preload property of the returned object is set to auto; the src property is set to the argument value. The browser begins asynchronously selecting the resource before returning the object.

mozSetup(in PRUint32 channels, in PRUint32 rate) Requires Gecko 2.0 Non-standard

- Sets up the audio stream for writing. You specify as parameters the number of channels (1 for mono or 2 for stereo), then the sampling rate (44100 for 44.1kHz, for example).
mozWriteAudio(in jsval data) Requires Gecko 2.0 Non-standard unsigned long Writes audio into the stream at the current offset. Returns the number of bytes actually written to the stream.
