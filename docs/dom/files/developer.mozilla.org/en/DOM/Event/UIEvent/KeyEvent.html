Order of key events

There are keydown, keypress and keyup events. For most keys, Gecko dispatches a set of key events like:

  1. keydown
  2. keypress
  3. keyup

If the pressed key is one of the following keys, the keypress event isn't dispatched:

  • Shift key
  • Control key
  • Alt key (Option key on Mac)
  • Cmd key on Mac
  • Meta key on Linux

And there are some keys which toggle LED state such as CapsLock, NumLock and ScrollLock. On Windows, these keys dispatch keydown and keyup events only. On Linux, they dispatch keydown, keypress and keyup events . On Mac, CapsLock dispatches only keydown event (cannot dispatches keyup event by the platform event model's limitation), NumLock on external keyboard (without Fn) dispatches keydown, keypress and keyup events. NumLock with Fn doesn't dispatch any events, and ScrollLock on external keyboard doesn't dispatch any events. The inconsistent behavior whether keypress event is dispatched or not is a bug, see bug 602812 .

Order of key events during auto-repeat on Firefox 4 or later version

When a key is held down for auto-repeat, the key events are dispatched by following order on Firefox 4 or later (Gecko 2.0 or later), this is according to DOM3 Events.

  1. keydown
  2. keypress
  3. keydown
  4. keypress
  5. ...
  6. keyup

However, on some GTK environments such as Ubuntu 9.4, auto-repeat dispatches native keyup event and Gecko cannot know the native key events are caused by auto-repeat or not. Therefore, on such platforms, the auto-repeat dispatches the key events by following order:

  1. keydown
  2. keypress
  3. keyup
  4. keydown
  5. keypress
  6. keyup
  7. ...

In this case, web applications cannot know whether each key event is dispatched by auto-repeat or not too.

Order of key events during auto-repeat on Firefox 3.6 or earlier version

On Firefox 3.6 or earlier (Gecko 1.9.2 or earlier), the key event order depends on the platform.

On Windows, same as Firefox 4.0 or later.

On Mac, only keypress events are repeated during auto-repeat such as:

  1. keydown
  2. keypress
  3. keypress
  4. ...
  5. keyup

On Linux, depends on the platform. If a platform doesn't dispatch native key up events during auto-repeat, the order is same as Mac.

Otherwise:

  1. keydown
  2. keypress
  3. keyup
  4. keydown
  5. keypress
  6. keyup
  7. ...
