DOMImplementation.createDocumentType

Table of contents
  1. 1. Syntax
  2. 2. Example
  3. 3.  Specification

Returns a DocumentType object which can either be used with DOMImplementation.createDocument upon document creation or they can be put into the document via Node.insertBefore() or Node.replaceChild(): http://www.w3.org/TR/DOM-Level-3-Cor...l#ID-B63ED1A31 (less ideal due to features not likely being as accessible: http://www.w3.org/TR/DOM-Level-3-Cor...createDocument ). In any case, entity declarations and notations will not be available: http://www.w3.org/TR/DOM-Level-3-Cor...-createDocType   

Syntax

doctype = createDocumentType (qualifiedNameStr, publicId, systemId)

qualifiedNameStr is a qualified name (like svg:svg), while publicId and systemId supply the PUBLIC and SYSTEM identifiers, respectively. A DocumentType object is returned.

Example

var dt = document.implementation.createDocumentType('svg:svg', '-//W3C//DTD SVG 1.1//EN', 'http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd');
var d = document.implementation.createDocument('http://www.w3.org/2000/svg', 'svg:svg', dt);
alert(d.doctype.publicId); // -//W3C//DTD SVG 1.1//EN
